# Revenue Analysis and Hypothesis Testing

## Project Goal
Analyze user behavior and revenue across different tariff plans and statistically verify whether there are significant differences between user groups.

---

## Data Description
The datasets includes:
- user information
- calls, messages, internet usage
- tariff plans

---

## Objectives
- Calculate user-level metrics (calls, messages, traffic)
- Estimate monthly revenue per user
- Compare tariff plans
- Test hypotheses about differences in user behavior and revenue

---

## Tools & Technologies
- Python (pandas, numpy)
- Data aggregation & feature engineering
- Statistical testing:
  - t-test
  - bootstrap
- Data visualization (matplotlib)

---

## What Was Done
- Cleaned and merged multiple datasets
- Aggregated user activity to monthly level
- Calculated revenue based on tariff rules
- Compared distributions across groups
- Conducted statistical hypothesis testing:
  - tested differences in revenue
  - validated assumptions
  - applied bootstrap for robustness

---

## Key Results
- Identified statistically significant differences between tariff plans
- Found differences in user behavior patterns
- Determined which plan generates higher revenue

---

## Skills Demonstrated
- Data preprocessing and merging multiple sources
- Feature engineering (user-level aggregation)
- Statistical hypothesis testing
- Revenue analytics
- Analytical thinking and interpretation of results
